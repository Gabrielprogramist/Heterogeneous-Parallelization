# Практическая работа №8

**Разработка гибридного приложения CPU + GPU (OpenMP + CUDA)**

## Цель работы

Разработать гибридное приложение, распределяющее вычисления между **CPU и GPU**, изучить принципы гибридных вычислений, передачу данных между CPU и GPU, а также проанализировать производительность различных режимов выполнения.

---


## Описание реализации

Вся практическая работа реализована **в одном исходном файле** `hybrid_cpu_gpu.cu`.

Это сделано **осознанно и обоснованно**, поскольку все задания практической работы логически связаны между собой и представляют **разные режимы обработки одного и того же массива данных**.

---

## Реализованные режимы обработки

### 1. Обработка массива на CPU (OpenMP)

* Используется многопоточность OpenMP.
* Каждый элемент массива умножается на 2.
* Время выполнения измеряется с помощью `std::chrono`.

Этот режим соответствует **Заданию 1**.

---

### 2. Обработка массива на GPU (CUDA)

* Массив копируется с CPU на GPU.
* Выполняется CUDA-ядро для параллельной обработки массива.
* Результат копируется обратно на CPU.
* Время выполнения измеряется с помощью `cudaEvent`.

Этот режим соответствует **Заданию 2**.

---

### 3. Гибридная обработка CPU + GPU

* Массив делится на две равные части.
* Первая половина обрабатывается на CPU с использованием OpenMP.
* Вторая половина обрабатывается на GPU с использованием CUDA.
* CPU и GPU работают **одновременно**.

Этот режим соответствует **Заданию 3**.

---

### 4. Анализ производительности

* Сравнивается время выполнения:

  * CPU-обработки,
  * GPU-обработки,
  * гибридной обработки.
* Результаты выводятся в консоль.
